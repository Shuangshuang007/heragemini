# 生产环境测试说明

## 问题
生产环境 https://www.heraai.net.au/api/mcp-lite 返回 404

## 原因
代码已提交到 GitHub，但 Vercel 可能还在部署中或需要手动触发部署

## 解决方案

### 方案 1: 等待自动部署（推荐）
1. 检查 Vercel Dashboard: https://vercel.com/dashboard
2. 查看最新部署状态
3. 等待部署完成（通常 1-3 分钟）
4. 重新运行测试脚本

### 方案 2: 手动触发部署
1. 在 Vercel Dashboard 中点击 "Redeploy"
2. 等待部署完成
3. 重新运行测试脚本

### 方案 3: 使用 Preview URL 测试
如果有 Vercel Preview URL，可以设置环境变量：
  TEST_URL=https://your-preview-url.vercel.app node scripts/test_mcp_lite_production.js

## 运行测试
部署完成后，运行：
  MCP_SHARED_SECRET=$(grep MCP_SHARED_SECRET .env.local | cut -d '=' -f2 | tr -d '"') node scripts/test_mcp_lite_production.js

## 测试内容
1. GET /api/mcp-lite - 工具列表发现
2. POST tools/list - 工具定义
3. 认证测试（无 Token、无效 Token）
4. Allowlist 检查（无效工具名）
5. 所有 4 个工具的调用测试
